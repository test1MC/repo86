Online vs. Print-Based Output

HTML5—Recommended Online Output Type

There are several online output options available in , but HTML5 is the recommended format.

PDF and Word—Recommended Print-Based Output Types

provides multiple print-based output options, but PDF and Word are recommended.

Other Output Types

Following are a few other unique output options in :

  • Clean XHTML See About Clean XHTML Output.

  • Eclipse Help This is a Java-based Help system that requires the latest Java Runtime Environment (JRE) and Java version. Use to create an Eclipse plug-in for the Eclipse Help Viewer. See About Eclipse Help Output.

    For more information, see the Eclipse Help Guide. For links to this PDF manual and others, see the online Help.

  • EPUB This is a format for creating reflowable digital books (ebooks). It is developed and maintained by the International Digital Publishing Forum (IDPF). EPUB is sort of a hybrid format. The output is designed to be viewed electronically; in that way, it is like an online format. On the other hand, its structure is like a book or manual; in that way, it is like a print-based format. In fact, you can follow many of the same steps for creating EPUB output that you would follow for producing a PDF. See About EPUB Output. For additional information about this group and the EPUB format, see http://idpf.org/.

HTML Help (Legacy Output)

In addition to the output options mentioned, you can create HTML Help, which is an older legacy online output not often used by most authors. See the online Help for information about this output format.
