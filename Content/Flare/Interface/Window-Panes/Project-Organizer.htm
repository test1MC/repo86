Project Organizer

The Project Organizer (along with the Content Explorer) is one of the most often used areas of the interface. Just as it sounds, the Project Organizer contains and organizes all of your project-related files.

How to Open the Project Organizer

  • Ribbon Select View > Project Organizer.

  • Keyboard Shortcut Press CTRL+[ on your keyboard.

What You Can Do in the Project Organizer

Following are the primary tasks that you can perform in the Project Organizer:

  • Auto Suggestion List Files (Add/Open) Read more… When you first start typing content in a topic, a popup may display below your text as you type. This is the Auto Suggestion feature, which analyzes text as you type and compares it to other content (e.g., phrases, variables, snippets) that already exists in the project. This means you can quickly select existing content from the popup so that you do not have to finish typing it. See Auto Suggestion and Adding Auto Suggestion List Files.
  • Browse Sequences (Add/Open) Read more… If you have several topics that you think end users should read in order, you can create browse sequences. When users view the compiled online Help system, they can use your browse sequences to "browse" through topics in a particular order. See Browse Sequences, Adding a Browse Sequence File, and Opening a Browse Sequence.
  • Conditions (Add/Open/Apply) Read more… A condition tag set is used to hold condition tags you create for your project. If you want to apply a condition to content, you can click and drag it from the Project Organizer to the content in the XML Editor. See Adding Condition Tag Set Files, Opening Condition Tag Sets, and Applying Conditions to Content.
  • Confluence—Import File (Add/Open) Read more… A Confluence import file is used to import Atlassian Confluence pages into a project. See Adding a Confluence Import File and Opening a Confluence Import File.
  • Context-Sensitive Help—Alias Files (Add/Open) Read more… An alias file is used to populate a header file with the information necessary for producing context-sensitive Help (CSH). In , you can open an alias file and use the Alias Editor to create and assign identifiers for the header file. See Context-Sensitive Help, Adding Alias Files, and Opening Alias Files.
  • Context-Sensitive Help—Header Files (Add/Open/Export) Read more… A header file is a simple text file that contains basic information about connecting the dialogs or windows in a software application to the corresponding topics in the Help system. Both you and the software developer need access to this file. See Context-Sensitive Help,Adding Header Files, Opening Header Files, and Providing a Developer With a Header File.
  • DITA—Import File (Add/Open) Read more… A DITA import file is used to import DITA file content into a project. See Adding a DITA Import File and Opening a DITA Import File.
  • Excel—Import File (Add/Open) Read more… An Excel import file is used to import Excel files into a project. See Adding an Excel Import File and Opening an Excel Import File.
  • Export File (Add/Open) Read more… An export file is used to export an entire Flare project, or parts of one, to another location. See Adding an Export Project File and Exporting Projects.
  • File Tag Sets (Add/Open) Read more… A file tag set is used to assign "tags" to topics and any other files in , even folders. You can use file tags for many different purposes, such as assigning authors or milestones to topics. lets you generate reports based on the tags that are assigned. This makes project development easier to track, manage, and schedule. See Adding File Tag Sets.
  • Flare Project—Import File (Add/Open) Read more… A project import file is used to import files from other projects. See Adding Project Import Files and Global Project Linking—Importing Files from Other Projects.
  • Folders (Create) Read more… When you add project files, they are added to various folders in the Project Organizer. As your list of files grows, it may become somewhat difficult to find a specific file when you want to work on it. In order to keep your files organized and easier to find, you can create subfolders in the Project Organizer and move files into them. See Creating Subfolders.
  • FrameMaker—Import File (Add/Open) Read more… A FrameMaker import file is used to import FrameMaker documents into a project. See Adding a FrameMaker Import File and Opening a FrameMaker Import File.
  • Glossaries (Add/Open) Read more… A glossary is a feature that you can add to your output to help users understand the meaning of individual terms. See Glossaries, Adding a Glossary File, and Opening a Glossary.
  • HTML—Import File (Add/Open) Read more… An HTML import file is used to import HTML files into a project. See Adding an HTML Import File and Opening an HTML Import File.
  • Indexes—Auto-Indexes (Add/Open) Read more… An auto-index lets you automatically add words and phrases in your project to a generated index, rather than inserting the index markers manually. In the Project Organizer, you can add and open an auto-index phrase set. See Creating Auto-Indexes.
  • Indexes—Link Sets (Add/Open) Read more… This is an index entry that links to another index entry. There are three kinds of index links: "See," "See Also," and "Sort As." In the Project Organizer, you can add and open an index link set. See Creating Index Links.
  • Language Skins (Add/Open) Read more… See Creating User-Defined Language Skins and Editing Language Skins.
  • Publishing Destinations (Add/Open) Read more… A publishing destination is a project file that contains the information needed for publishing output (i.e., copying the files) to another location (such as a website or a directory on your computer). See Adding Destination Files and Opening Destinations.
  • Relationship Tables (Add/Open) Read more…  See Relationship Tables, Adding Relationship Tables, and Opening Relationship Tables.
  • Reports (Add/Open) Read more…  See Analysis Ribbon.
  • Search—Filter Sets (Add/Open) Read more… A search filter lets users narrow their search based on concepts that you have inserted into topics. Concepts are simply keyword markers that you insert into topics that have some kind of relationship with each other. They are also used for inserting concept links into topics. See Search, Adding Search Filter Set Files to Projects, and Opening a Search Filter Set.
  • Search—Synonym Files (Add/Open) Read more… A synonym file lets you create synonyms for terms in your project so that users are more likely to find the information they need when performing searches. See Search, Adding Synonym Files, and Opening Synonym Files.
  • Skins (Add/Open) Read more… A skin is a file that contains information about the appearance of the output window. See Skins, Adding Skins, and Opening Skins.
  • Tables of Contents (Add/Open) Read more… In many cases, provides you with an initial table of contents (TOC), which you further develop using the TOC Editor. See Tables of Contents, Adding a TOC File, and Opening a TOC.
  • Targets (Add/Open) Read more… A target is one instance of an output type. When you build your final output, you are essentially building one or more of the targets in your project. You can add regular targets, as well as batch targets. See Adding Targets, Building Output Using a Batch Target, and Opening Targets.
  • Variables (Add/Open/Insert) Read more… A variable set is a file that holds one or more variables. You can expand the variable sets in the Project Organizer to see the variables within them. If you want to insert a variable into a document, you can click and drag it from the Project Organizer to the XML Editor. See Variables, Adding Variable Sets to Projects, Opening a Variable Set, and Inserting Variables.
  • Word—Import File (Add/Open) Read more… A Word import file is used to import Microsoft Word documents into a project. See Adding an MS Word Import File and Opening an MS Word Import File.

You can right-click on files in the Project Organizer and perform tasks from the context menu that displays. For example, you can select "Open with" and open the selected file in another application (e.g., Notepad++).

Buttons and Sections

Toggles between showing or hiding the folders in the Project Organizer.

Splits the Project Organizer into two halves. When you select a folder on the left side, the folders and files within it are shown on the right side. Also, you can select multiple files on the right side, which lets you perform the same action on all of them at the same time.

Conditional Text

Holds the condition tag sets that you add to the project. You can expand the condition tag sets to see the conditions within them, and you can drag those conditions to content in the XML Editor. See Conditions.

Destinations

Holds the publishing destinations that you add to the project. See Creating Publishing Destinations.

Glossaries

Holds the glossaries that you add to the project. See Glossaries.

Exports

Holds export files that you add to the project. These are used to export your Flare project (or parts of it) to another location. See Adding an Export Project File.

Imports

Holds import files that you add to the project. See Importing.

Reports

Holds reports that you create in order to track information in your project. See Adding Report Files.

Skins

Holds the skins and skin components that you add to the project. See Skins.

Targets

Holds the targets that you add to the project. You can add as many targets to a project as you need. In addition to regular targets, you can add batch targets, which are used to generate multiple targets in one batch. See Step 4: Developing Targets.

TOCs

Holds the tables of contents that you add to the project. See Tables of Contents.

Variables

Holds the variable sets that you add to the project. You can expand the variable sets to see the variables within them, and you can drag those variables to content in the XML Editor. See Variables.

[Preview Bar]

This is the area at the bottom of the Project Organizer. When you click on a file or folder, this bar shows an icon, as well as the name of the folder or file.

If you have a project file open and want to quickly find where it is stored in the Project Organizer, you can right-click the tab of the file and select Locate in Explorer. You can also open the Project ribbon and click Locate in Explorer. Flare opens the Project Organizer, expanding folders to show where the file is located.

You can drag and drop files from Windows into the Project Organizer. See Dragging and Dropping Files Into Projects.
