Creating Destination Files Using Source Control

You can create a destination file using Source Control if you want to publish output files to a source control server, such as TFS or Subversion.

How to Create a Destination File Using Source Control

  3. In the Destination Editor, click in the Type drop-down field, and select Source Control.
  4. In the Directory field, enter the folder where the output files will be published. You should enter a location that is not your project's output folder. This is because you do not want to inadvertently overwrite or delete your published files.
  5. (Optional) You can the Comment field to provide a description of the destination so that its purpose is clear.
  6. Complete the Source Control section.

    1. Create or detect a source control binding.

      If Your Project Is Already Bound to Source Control

      • If you are creating a destination file in order to publish to source control, you can perform a bind detection if your project is already bound to source control. Click Detect to search for existing source control bindings on the current destination folder. If they exist, Flare will automatically fill in the fields using this information.

        Source control providers are scanned in the following order:

        1. Git
        2. Subversion

        3. Perforce Helix Core

        4. Microsoft Team Foundation Server (TFS)

        To Set a Bind Destination—TFS

          1. To add a server, click . The Add Team Foundation Server dialog opens.
          2. Enter the name or URL of the server.
          3. Enter the path and port number.

          4. Select the protocol (HTTP or HTTPS).

          Click OK in the dialogs until you return to the main wizard page.

          Be sure to specify the path deeper than the server name. For example:

          .../tfs/DefaultCollection

          or

          .../tfs/SourceCollection

          If you do not know this information, contact your system administrator.

          If you run into issues in this dialog, chances are your user permissions do not have access to all of the collections on the TFS server. Please try to enter in the server/collection path manually.
