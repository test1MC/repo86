Troubleshooting XML Editor Issues

Following are issues that may occur when using the XML Editor in Flare.

  • When saving a topic in the XML Editor, a message appears saying that the topic has been updated outside of the editor
  • Flare's "spell check while typing" feature is malfunctioning

When saving a topic in the XML Editor, a message appears saying that the topic has been updated outside of the editor

When saving a topic in the XML Editor, the following message appears:

[Topic Name and location] has been changed outside the editor.

Do you want to discard the changes and reload?

Cause

This issue is caused when there is an unsaved topic in the XML Editor and changes are made to the same topic in an interface outside of the XML Editor. For example, if you access topic properties from the Content Explorer and change properties on the unsaved topic (e.g., file tags, language settings, the topic name, topic style class) and then click OK, the message will appear. Flare recognizes that changes have been made to the topic outside of the XML Editor and prompts you with this message to decide whether you want to discard changes and reload the topic.

Selecting "yes" to the "discard the changes and reload" option will result in any changes made to the topic in the XML Editor to be discarded. The topic will then be saved. The changes made to the topic outside of the XML Editor will also be saved. Selecting the "no" option will save the changes in the external interface but leave the XML Editor changes unsaved.

Solution

If you want to avoid this dialog, simply save topic in the XML Editor before you make changes to a topic in an interface outside of the XML Editor.

For more information see XML Editor.

Flare's "spell check while typing" feature is malfunctioning

The "spell check while typing" function in the XML Editor is flagging all of the text as misspelled.

Cause

This problem can be caused by having the incorrect language set on the project or topic.

Solution

Make sure that the correct language is set on the project or topic.

how to select a language for the entire project

  1. Select Project > Project Properties. The Project Properties dialog opens.
  2. Select the Language tab.
  3. From the list, select a language (those in bold font are supported in spell check).

how to select a language for a specific topic

  1. Find and select the topic.
  2. In the local toolbar of the Content Explorer, click . The Properties dialog opens.
  3. Select the Language tab.
  4. From the list, select a language (those in bold font are supported in spell check).
