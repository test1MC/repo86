Troubleshooting Condition Tag Issues

Following are issues that may occur when using condition tags in Flare.

  • New condition tags have been set on content but the new tag is not shown
  • .NET unhandled exception error occurs when attempting to include or exclude a condition tags

New condition tags have been set on content but the new tag is not shown

Cause

This issue can occur because the window pane or editor needs to be refreshed.

Solution

Refresh the window pane or editor by bringing it into focus and pressing F5 on the keyboard.

.NET unhandled exception error occurs when attempting to include or exclude a condition tags

A Microsoft .NET Framework unhandled exception error when attempting to include or exclude a condition tag on a target with a non-alphanumeric character in its name.

Cause

This is caused because Flare does not currently support non-alphanumeric names for condition tags. There is currently a bug in Flare's Condition Tag Set Editor which lets you name condition tags with non-alphanumeric characters. The exception error occurs when an attempt to set those conditions on a target in the Target Editor is made.

Solution

Refrain from naming conditions in the Condition Tag Set Editor with non-alphanumeric characters until the problem is addressed in a future version.
