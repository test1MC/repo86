Windows 7 causes Adobe FrameMaker import to fail

The following error appears when trying to import Adobe FrameMakerfiles into Flare:

"Failed to run command 'Add FrameMaker Import File …'"

Exception: Failed to update FrameMaker init file (C:\Program Files(x86)\Adobe FrameMaker 7.2\maker.ini)

Cause 

Failure to import Adobe FrameMaker files into Flare might be a result of Windows 7 and its use of User Account Control (UAC). If this is the case, performing the following steps allows Flare to write to the file.

Solution

  1. Locate the maker.ini file (its location is listed on the message). For example, from the error shown above, you would use the Windows Explorer to navigate to C:\Program Files(x86)\Adobe FrameMaker 7.2\.

  2. Right-click on the file called "maker.ini" and from the context menu, select Properties.

  3. Select the Security tab.

  4. Select Edit.

    The Permissions for maker.ini dialog opens.

  5. Select your Windows log-in user name.

  6. Make sure that the Full Control box is checked.

  7. Click Apply.
  8. Click OK to exit the Properties dialog. You should now be able to import successfully.

There are missing arrowheads on lines in SVG images imported from MS Visio

Cause

The reason for this problem is that Microsoft Visio fails to write the style correctly, causing the arrowheads on lines to disappear.

Solution

To solve this problem, perform the following steps.

  1. Open the SVG file in Notepad.
  2. Select Edit > Replace.

  3. In the Find field, enter the following.

    orient="auto"

  4. In the Replace field, enter the following.

    orient="auto" style="overflow:visible;"

  5. Open the file in a web browser to verify the arrowheads are now part of the image.
  6. Import the image into Flare.

Import error when importing microsoft word documents into Flare

When importing projects into a Flare project, Flare displays the following error message:

Error converting the following document:

[document name]

The document can not be imported to Flare unless Microsoft Word can save the file in XML format.

Cause and Solution

This error can have the following causes.

cause/solution 1

This error appears because the document being imported has its attribute set to "Read-only". Flare cannot save a document in XML format when its attribute is set to "Read-only", and therefore cannot import it into a project.

Set the attribute of the document being imported to "Read-only". Perform the following steps.

  1. In the Windows Explorer, navigate to the location of the file that you want to import into Flare.
  2. Right-click the file and from the context menu, select Properties.
  3. In the Attributes section, click the Read-only check box so that it is unchecked.
  4. Click Apply.
  6. Import the document into Flare. The error will not appear.

cause/solution 2

The macros set in the document are causing problems with the import process.

Disable the macros in the document.

cause/solution 3

The document needs to be converted to XML format.

Save the document as an XML document, then save it again as a document.

  1. In , select File > Save As.
  2. In the Save as type field, select Word XML Document.
  3. Click Save.
  4. Repeat steps 1 through 3, but in the Save as type field, select Word Document.
  5. Open Flare and import the document. The error should not appear.

For more information see Importing Word Files.

When importing from , artwork and image effects do not come across

Cause

If you apply certain special effects or artwork (e.g., arrows) to images in Word and then import them into , those effects will not be retained. For example, a rotated image will return to its straight, original state. However, there are some workarounds (listed below) that may allow you to keep the effects or artwork. As for text floating around images, this effect is not supported in . When you import from Word, the text is added below the image.

Solution

Try the following.

solution 1

Apply another effect such as a shadow or 3-D to the image in Word (e.g., add a shadow to a rotated image that you want to keep). This forces Word to save the image as an entirely new image with the effects.

solution 2

  1. Save the Word document as a web page (HTML file).
  2. Manually copy that HTM file into the folder.
