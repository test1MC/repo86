Enabling Feedback in Flare

See About MadCap Feedback.

How to Enable Feedback in Flare

  1. From the Project Organizer, open the target.
  2. Select the Analytics tab.
  3. From the Provider field, select Pulse/Feedback Server.

  4. In the URL field, enter the path to the server hosting the data.

    It is highly recommended that you purchase an SSL certificate and install it on your web server. SSL uses encryption for certain processes, which helps to ensure that the data transfer is secure. If you have an SSL certificate installed on your web server, make sure you begin the path to the server with https://.

  5. Copy the Feedback license that you have been issued and paste it into the Feedback Service License Key field.

How to Set the Community Options (Display Feedback Comments) Using a Stylesheet

  1. Open the stylesheet.
  2. Find and select the html style or the appropriate class.
  3. In the Show filter field, select Alphabetical List.
  4. Find the mc-community-features property and set it to enabled or disabled. The default setting is "enabled."

This stylesheet method can be especially useful if you only want some topics to display the comment area.

If there is a discrepancy between the skin and the stylesheet setting (e.g., the skin is set to disabled and the CSS is set to enabled), the stylesheet setting has precedence.

Comments at the bottom of topics are not shown in tablet or mobile view. This is the same case for HTML5 Tripane output.

The comments area at the bottom of topics is wrapped in an iframe. Therefore, the output is technically not entirely frameless. However, because it is only the comments that are contained in the iframe, the main topic content still retains the benefits (e.g., better search results) of frameless output.

After you enable Feedback in a Flare target, you can do the following.

  • Configure the target with Feedback Explorer so that you can view and work with user activity for that published target. See Configuring a Target with Feedback Explorer.
  • Disable comments in the target skin (if you do not want users to be able to submit and view comments). See Disabling Feedback User Comments in Skins.

After you generate and publish the output, you can open Feedback Explorer to see how readers are using the output. See Opening Feedback Explorer.
