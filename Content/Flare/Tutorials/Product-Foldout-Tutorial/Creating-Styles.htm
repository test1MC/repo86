Creating Styles

The page layouts, topics, and TOC have been created. And you've added all of the content to the topics. However, the content still needs to be formatted with styles.

All of the styles you need already exist in the stylesheet in the template. But we're going to show you how we created a couple of the styles so you can see how it is done.

How to Identify Styles

Before we create new styles, we want to show you how to identify existing styles. If you've ever looked at a section of content and thought, "I wonder how they got it to look like that," you can easily find out. It's just a matter of knowing where to look.

  1. Open the Front2.htm topic.
  2. In the first panel, click somewhere in the blue heading.

  3. Look to the far left side of the XML Editor and notice the structure bars.

    The structure bar that is related to the heading has an h1 label.

  4. Click somewhere in the line below that contains the two variables.

  5. Now look above the topic. You should see a span bar. If your span bars are not turned on, click at the bottom of the editor.

    Span bars represent elements and formatting that is done inline. Since the current paragraph has a couple of variables, you will see span bars at the top with MadCap:variable labels.

    Similarly, if the paragraph had any bold text, you would see a bar labeled “b.” And if it had a unique character-level style, you would see a bar simply labeled “span,” but you can hover over the bar to see the exact style name.

  6. Let's investigate the h1 style. Right-click the h1 structure bar and select Style Class > Edit Style Class.

    The stylesheet opens and automatically selects the h1 style. If it doesn't automatically select it, go ahead and select h1.

  8. In the local toolbar, click . This changes the view of the properties on the right from groups to an alphabetical list.

  9. On the right side of the local toolbar, click the Show drop-down field and select Show: Set Properties. This filters the properties below to only those that have been explicitly set in the stylesheet for the selected style, as well as any properties explicitly set in an inherited stylesheet.

    As you can see, multiple properties were set on this style, including its size, color, and margins. Any property shown in gray is being inherited from somewhere else (in this case, a factory stylesheet where you installed Flare).

How to Create Styles

Let's re-create a style class that is used in this template. If you look at the original Front topic, you will see that the first paragraph under the heading is designed to have a larger font size and line height than a normal paragraph.

We are going to intentionally create a duplicate of that style class, which we will apply to our Front2 topic.

  2. On the left side of the Stylesheet Editor, select the p style.

  3. In the local toolbar, click Add Selector.

  4. In the New Selector dialog, click in the Class Name field and type the following:

    large-new

  5. The new class is added under the p style and is selected.

  6. On the right side of the local toolbar, click in the Show drop-down field and select Show: All Properties. This displays all of the available properties below.

  7. On the right, scroll down the list of properties until you find each of the following properties. Then to the right of each one, click and give them the values shown:
    • font-size 13pt
    • line-height 15pt
