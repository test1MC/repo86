Enabling Glossaries in Skins

After you create glossary terms and definitions, you need to enable glossaries in the skin that you intend to use for your target. This task is necessary only for some online outputs. For HTML5 Side and Top Navigation skins, this task is not necessary because those outputs do not use a Glossary window pane in the output.

How to Enable a Glossary in a Skin

  1. From the Project Organizer open a Standard or HTML5 Tripane skin (not a Side Navigation or Top Navigation skin).
  2. On the General tab, click the Glossary check box so that it contains a check mark.

After you have finished enabling glossaries in a skin, you need to make sure the skin is associated with a target and the glossary is associated with a target. It is at this point that you can specify whether certain occurrences of terms in topics should be automatically converted to links. See Associating Skins with Targets and Associating Glossaries with Targets.

The conversion of a glossary term to a link is case-sensitive. For example, if a term starts with an uppercase letter in the glossary, it will not be converted to a link in the output of a topic if the term is all lowercase in the topic.
