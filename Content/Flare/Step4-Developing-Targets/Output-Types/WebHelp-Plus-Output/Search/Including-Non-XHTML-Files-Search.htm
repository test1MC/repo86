Including Non-XHTML Files in WebHelp Plus Search

 

How to Include Non-XHTML Files in Search

  1. Enable Output See Enabling WebHelp Plus Output. 
  2. Generate Output In Flare, generate a target using the output format. See About Building and Publishing Output.
  3. Publish Output to Server
  4. (Optional) Add IFilter

  1. Copy and Paste Non-XHTML Files to Appropriate Folder

If you are testing output on your local machine, you need to view the output at least one time. When testing this search feature on your computer, you need to place the non-XHTML files in the AutoSearch subfolder at this location (as opposed to the output folder that was generated where your Flare project is located).

The information just provided revolves around output. However, you can also perform these same steps for HTML5 output if you have enabled the server-based option. For more information see the online Help.
