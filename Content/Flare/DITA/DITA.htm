Why Use DITA?

DITA is not for everyone (e.g., smaller companies, individual writers). But if you are in need of very tightly structured content solution, DITA might be something to consider.

DITA Issues

There are at least a couple of questions you need to answer if you have a need for DITA.

First, how do you write your content so that it is using the DITA schema?

As DITA takes on a bigger role in technical communication, more and more tools are becoming available to create this type of structured content. In this version, you cannot use to edit DITA content natively. However, you can use a third-party tool (e.g., XMetal) to create your structured DITA files.

A second major question is this: once you have created DITA files, how do you get them into a useful output for end users?

Although there are a few tools for creating DITA content, there is a big problem when you need to generate useful online and print-based output. Writers often have difficulty producing a custom look and feel for the DITA output or taking advantage of single-sourcing (e.g., generating both online and print output from the same set of files). This can result in the need for additional open tool kits or Java runtime. It can mean additional development resources to create custom transforms or other custom programming.

This is where comes in. After you create your DITA content (regardless of the tool), you can import content from those DITA files into . You then have at your disposal all of the single-sourcing features and the necessary tools to generate the type of output(s) you need, without having to use additional resources or programmers.
