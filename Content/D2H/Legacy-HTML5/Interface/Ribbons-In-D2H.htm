Ribbons

There are several ribbons in the Doc-To-Help application that are used to assist you in managing your Doc-To-Help project.

Ribbons in Doc-To-Help

Ribbons in Doc-To-Help

Doc-To-Help has multiple ribbons that are used for various purposes. The ribbon layout consists of a File drop-down with common tasks such as creating and opening new projects, as well as accessing unique tools and options. This is followed by the Home ribbon, which is the default, and other ribbons that are related to various features in .

Each ribbon is divided into logical groups. Many dialogs can be opened directly from ribbon groups using dialog launchers. (The dialog launchers are the small arrows on the bottom-right of specific ribbon groups.)

You can minimize the Ribbons using the Quick Access Toolbar option, which can be accessed from the down arrow located above the ribbons. See Quick Access Toolbar.

For other navigation methods, see Touring the Workspace.

File Drop-Down

New Project

Create or open a project. See Creating and Converting Projects. See the online Help or the Getting Started Guide.

Open Project

Opens a dialog, which lets you open any type of file supported by .

Save Project As

Tools

Access the Tools menu, where you can do the following:

  • Set options (startup options, default editors, and more). See Options Dialog.
  • Deactivate Doc-To-Help.
  • Check for updates.
  • Compact your project, reducing its file size. See Compacting a Project File. See the online Help or the Project Management Guide.
  • Import and export project settings. See Importing and Exporting Project Settings. See the online Help or the Project Management Guide.
  • Access the build scheduler. See Scheduling Builds with the Build Scheduler. See the online Help or the Targets Guide.
  • Convert older DOC files to DOCX. See Converting DOC Files to DOCX. See the online Help or the Documents, Templates, and Content Guide.

Reports

Doc-To-Help has several reports that will help you organize your work and keep track of topics and index elements. These reports can be viewed, saved, and printed for maximum flexibility. See Reports. See the online Help or the Project Management Guide.

Team Authoring

Team authoring is a source control feature that makes it possible for multiple authors to work together on a single project. Authors work on their own local copy of a project on their machine (called the working copy), while the parent project (or the team project) is located on the organization's network or on a web server. Although the same project is made available to the entire team, one author's changes will not be overwritten by another author's changes. See About Team Authoring. See the online Help or the Project Management Guide.

 Home Ribbon

Select Target

Select the target using the drop-down list. The icon above it will display the target type. See About Targets. See the online Help or the Targets Guide.

Build

Build the current target. See Building Targets. See the online Help or the Targets Guide.

Rebuild

Rebuild the current target. When rebuilding a Help target, Doc-To-Help empties its output directory and recompiles the entire project. See Building Targets. See the online Help or the Targets Guide.

View

View the current Help target output.

View PDF

If you have a Manual target type selected and have it configured to also generate a PDF (in the Help Targets dialog), you can click this button to view the generated PDF.

Publish/Publish to SharePoint

Publish the NetHelp Target to a SharePoint Document Library or SharePoint Wiki (only visible if a NetHelp target has been selected). See Publishing to SharePoint. See the online Help or the Project Management Guide.

Cancel Build

Cancel the current build.

Build Log

View the Output Window, which displays the build log, as well as errors and unresolved links.

Theme

Select and edit the theme (skin) for the current help target. Click this button to open the Theme Preview. See About Themes. See the online Help or the Design Guide.

Target Template

This template controls the formatting of the target for Word source documents. Target template styles will override any matching styles in the Source template. See About Templates and Styles. See the online Help; the Documents, Templates, and Content Guide; or the Design Guide.

New

Add a new Word document to the project. See Adding, Renaming, and Removing Documents. See the online Help or the Documents, Templates, and Content Guide.

Add

Add an existing Word document to the project. See Adding, Renaming, and Removing Documents. See the online Help or the Documents, Templates, and Content Guide.

Remove

Remove the selected document from the project.

Platforms

Set a platform-based condition for the selected document. The document will be included in all of the target platform(s) selected. A platform includes all the targets that produce the output specified; Online Help, HTML Help, NetHelp, Printed Manual, etc. See About Conditions. See the online Help or the Documents, Templates, and Content Guide.

Targets

Set a target-based condition for the selected document. The document will be included in all the target(s) selected. See About Conditions. See the online Help or the Documents, Templates, and Content Guide.

Attributes

Set an attribute-based condition for the selected document. The document will be included or excluded when creating conditional builds (e.g., internal or external.) Use the Attributes dialog to create custom attributes. See About Conditions. and Creating and Using Attributes. See the online Help or the Documents, Templates, and Content Guide.

Source Template

The default template for new Word source documents added to the project. To change the look of the output, edit the target template. Styles that exist only in the source template will pass through to the target template. See About Templates and Styles. See the online Help; the Documents, Templates, and Content Guide; or the Design Guide.

Share

Upload documents to a Microsoft® SharePoint® Library. See About SharePoint Integration. See the online Help or the Project Management Guide.

Synchronize

Synchronize shared documents that were changed either locally, or in the Microsoft SharePoint Library.

Work Offline

In Offline Mode, documents are disconnected from the SharePoint Library so Doc-To-Help will not check for updates and suggest synchronization automatically.

Related Topics

Platforms

A platform includes all the targets that produce the output specified; Online Help, HTML Help, NetHelp, Printed Manual, etc. See About Conditions. See the online Help or the Documents, Templates, and Content Guide.

Targets

Set a target-based condition for the selected topic. The topic will be included in all the targets selected. See About Conditions. See the online Help or the Documents, Templates, and Content Guide.

Attributes

Set an attribute-based condition for the selected topic. The topic will be included or excluded when creating conditional builds (e.g., internal or external). See About Conditions and Creating and Using Attributes. See the online Help or the Documents, Templates, and Content Guide.

Related Links Label

This is the label that will appear above related (subtopic) links for the selected topic. See Managing Related Topics. See the online Help or the Documents, Templates, and Content Guide.

Hide subtopic links

If this check box is selected, all subtopic links will be hidden in the selected parent topic. See Managing Related Topics. See the online Help or the Documents, Templates, and Content Guide.

Properties

Opens the Topic Properties dialog for the selected topic. See Viewing/Changing Topic Properties. See the online Help or the Documents, Templates, and Content Guide.

Open

Project Ribbon

Attributes

Opens the Attributes dialog where you can add or edit project attributes. Use these attributes to tag content for conditional builds. Attribute conditions can be applied to text, topics, and/or entire documents. See About Conditions and Creating and Using Attributes. See the online Help or the Documents, Templates, and Content Guide.

Windows

Opens the Windows dialog, where you can edit the default window types. Custom windows can also be created. See Setting the Window Display. See the online Help or the Design Guide.

Project Styles

Opens the Project Styles dialog, where you can customize the source paragraph and character styles. These styles are from the source template. You can also add new styles. See About Project Styles and Topic Types. See the online Help or the Design Guide.

Topic Types

Opens the Project Styles dialog, where you can add or edit topic types. A topic type is assigned to each topic. You can customize the default topic types or create new ones. See About Project Styles and Topic Types. See the online Help or the Design Guide.

Variables

Opens the Variables window, where you can create and edit plain and rich content variables. See About Variables. See the online Help or the Documents, Templates, and Content Guide.

Under the Variables button, click this to open the Project Settings dialog. See Setting Project Properties. See the online Help or the Project Management Guide.

Auto-Index

Automatically create the index based on one or more topic types or styles. Open the drop-down menu to set preferences. See About Indexes. See the online Help or the Documents, Templates, and Content Guide.
