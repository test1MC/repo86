Opening the Build Log

In Flare, the output files and build log are retained for only the most recent build for a target; when you rebuild, the previous output files and build log are replaced with the new ones. But in Central, the output files and build log are retained for each build.

How to Open a Build Log

  5. In the toolbar, click . The Build Log dialog opens.

Viewing and Downloading the Log File

The log file contains the following sections, which can be accessed by clicking them on the left:

  • Messages

  • Warnings

    Use caution when ignoring compiler warnings. Only ignore warnings that you are certain you do not need to receive.

    When you generate a target from Flare, you have the option of ignoring warnings globally (using the Options dialog) or only for a specific target (using the Target Editor). However, when generating a target from Central, you must set the warnings to ignore in the Target Editor; Central will not use any global settings from the Options dialog in Flare.

  • Errors

If you want a local copy of the log, click Download Log.

You can also view and download a build log from the Build Profile dialog. See Opening the Build Profile Dialog.
